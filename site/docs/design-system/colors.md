# Colors

The Kearney Design System (KDS) color palette follows a strict brand principle:

**"White + slate + purple accent"**

## Brand Constraints

### Approved Palette Only

All designs must use **only** the 27 approved colors listed on this page. The palette consists of:

- **Foundation**: White (#FFFFFF), Black (#000000), Charcoal (#1E1E1E)
- **Greys**: Complete scale from #F5F5F5 to #1A1A1A (14 shades)
- **Purples**: Primary scale (#7823DC to #E6D2FA) + secondary scale (#8737E1 to #D7BEF5)

### Forbidden Colors

**NO green, red, orange, or blue** are permitted anywhere in the design system.

This includes:
- ❌ Green for "success" or positive states
- ❌ Red for "error" or negative states
- ❌ Orange for "warning" or attention states
- ❌ Blue for "info" or neutral states

Instead, we use **purple for emphasis** and **greys with text labels** for semantic states.

## Semantic Tokens

Semantic tokens map to specific UI purposes and automatically adapt to light/dark themes.

### Why Not "Success" / "Error" / "Warning"?

Kearney brand does **not** use color-coded semantics. Instead:

- **`positive`** → Purple (not green) with text label "Improved" or ▲ symbol
- **`negative`** → Grey (not red) with text label "Declined" or ▼ symbol
- **`neutral`** → Grey (not orange) with text label "No change" or ─ symbol

**Color alone should never convey meaning.** Always pair color with text or symbols.

## Usage Guidelines

### Chart Colors

For **single-series charts** (timeseries, area, line):
- Use `chartHighlight` (#7823DC light, #C8A5F0 dark) for the main data series
- Use `chartMuted` for reference lines or secondary data

For **multi-series charts** (categorical bars, grouped data):
- Use the categorical palette: `[purple, silver, violet4, grey600, violet3, grey200]`
- Alternate between purples and greys for visual separation

**NO gridlines** in charts per Kearney brand. Use axis labels and data labels instead.

### Semantic Color Migration

If migrating from old semantic colors:

| Old Token | New Token | Notes |
|-----------|-----------|-------|
| `success` | `positive` | Use with ▲ or "Improved" label |
| `successTint` | `positiveTint` | Light purple tint (not green) |
| `error` | `negative` | Use with ▼ or "Declined" label |
| `errorTint` | `negativeTint` | Grey tint (not red) |
| `warning` | `neutral` | Use with ─ or "No change" label |
| `warningTint` | `neutralTint` | Grey tint (not orange) |

### Accessibility

**Color alone is not sufficient for conveying information.** Always pair color with:
- Text labels ("Improved", "Declined", "No change")
- Symbols (▲, ▼, ─)
- Icons or patterns

This ensures accessibility for colorblind users and compliance with WCAG 2.1 AA standards.

## Implementation

### CSS Custom Properties

```css
/* Use semantic tokens, not hardcoded values */
.button-primary {
  background-color: var(--emphasis);
  color: var(--text-inverse);
}

.button-primary:hover {
  background-color: var(--emphasis-hover);
}
```

### TypeScript / JavaScript

```typescript
import { colors, CATEGORICAL_PRIMARY } from '@/design-system/tokens';

// Use token constants
const primaryButton = {
  backgroundColor: colors.emphasis,
  color: colors.textInverse,
};

// Use categorical palette for charts
const chartConfig = {
  colors: CATEGORICAL_PRIMARY,
};
```

### React Components

```tsx
import { useTheme } from '@/contexts/ThemeContext';

function KPICard({ value, trend }) {
  const { theme } = useTheme();

  return (
    <div style={{ backgroundColor: theme.surface }}>
      <span style={{ color: theme.text }}>{value}</span>
      {/* Use symbols, not color alone */}
      <span style={{ color: theme.textMuted }}>
        {trend > 0 ? '▲' : trend < 0 ? '▼' : '─'}
      </span>
    </div>
  );
}
```
